Investigación Múltiple

Diferencias, implicaciones visuales e implicaciones en motores de búsqueda entre etiquetas

<b></b> vs <strong></strong>

Ambas etiquetas se utilizan para aplicar formato de negrita al texto y no existe una diferencia en cómo se ven en el documento. Sin embargo, la etiqueta <b> se utiliza principalmente con fines estilísticos, es decir, que sólo le dice al navegador que resalte en negritas el texto, mientras que la etiqueta <strong> indica texto con gran importancia dentro de la estrcutura del documento, pues se encarga de transmitir significado más allá de solo ser estético. Cuando se realiza una búsqueda en google, esta diferencia es la que hace que buscadores como google le den más peso al contenido dentro del <strong>.

<i></i> vs <em></em>

Pasa lo mismo con estas dos etiquetas, ya que ambas muentran el texto en itálicas y no hay una diferencia visual en el documento. Pero <i> es sólamente visual, mientras que <em> tiene un valor interpretativo de relevancia, por lo que google lo puede interpretar como un texto con mayor importancia dentro de la página.

<s></s> vs <del></del>

No es excepción con estas últimas dos etiqueticas, ambas muestran el texto tachado con una línea horizontal en el centro de la palabra y visualmente se ven iguales. La diferencia está en la conotación de cada una, ya que <s> es una etiqueta que especifica texto que ya no es correcto, relevante o preciso, mientras que <del> indica texto que ha sido eliminado del documento real. Por lo que nuevamente, <s> termina siendo visual y ante los buscadores será solamente un formato, mientras para los buscadores la etiqueta <del> significa que ese fragmento ya no forma parte del contenido principal.

Categorías a elementos o etiquetas

Elementos de línea

Un elemento de línea en HTML solo ocupa el ancho de su contenido y permite que otros elementos se encuentren en la misma línea. Se utilizan para dar formato a fragmentos pequeños dentro del texto. Unos ejemplos de estos elementos son:
  • <a>
  • <img>
  • <strong>
  • <i>
  • <em>

Elementos de bloque

Un elemento de bloque en HTML es el que ocupa todo el ancho de la página o del elemento que lo contiene y siempre empiezan en una nueva línea. Se usan para estructurar la página y organizar las secciones. Algunos ejemplos de este tipo de elementos son:
  • <p>
  • <h1> hasta <h6>
  • <div>
  • <footer>
  • <header>
Aunque solo pongas una palabra dentro de estos elementos, estos ocuparán todo el ancho disponible y como su nombre los dice "Elementos bloque", se comportan como una caja en donde se estrctura el contenido. Ejemplo visual:

Campos en los formularios

<input>

Este elemento se utiliza para crear controles en formularios web que aceptan datos del usuario. Algunas de sus propiedades son:
  • required: se usa para que sea obligatorio compleyar el campo antes de enviar
  • name: es el nombr que se le pone al campo
  • placeholder: es el texto que se le pone de guía al campo
  • accept: sólo permite los tipos de archivos especificados en el apartado
  • type: indica el tipo de dato en el formulario
Todavía hay muchos más. En cuanto al type hay muchos usos distintos, por ejmeplo:
  • text:es una caja de texto normal
  • email: es una caja que valida que el correo ingresado sea váido
  • number: es una caja de texto que solo acepta números
  • password: oculta el texto que el usuario escribe para mantener la seguridad
  • date: muentra un selector de calendario
  • file: permite seleccionar archivos desde la compu
  • range: muestra una barra deslizable para elegir un valor

<label>

Es un elemento en una interfaz de usuario. Sirve para asociar una etiqueta con un campo de netrada en un formulario, lo que ayuda a los usuarios a comprender la información solicitada en el mismo campo. Algunas de sus propiedades son:
  • for: es el id del elemento de formulario al que está asociado
  • form: lo asocia a un formulario específico mediante su id

<output>

Se usa en formularios para mostrar el resultado de un cálculo, como resultado de una acción en un formulario. Sus spropiedades son:
  • for: indica qué campos están relacionados con el resultado
  • form: lo asocia a un formulario específico mediante su id
  • name: es el nombre del elemento

<progress>

Se utiliza para visualizar el progreso de una tarea. Propiedades:
  • max: indica la cantidad de trabajo que demora la carga
  • value: indica qué parte de la tarea ya se ha completado

<select>

Crea una lista desplegable de opciones para que el usuario elija. Propiedades:
  • diseable: especifica si el usuario puede interactuar con el control
  • multiple: indica si se pueden seleccionar múltiples ocpiones
  • form
  • name
  • required

<option>

Define cada ocpión dentro de un select y representa un valor que el usuario puede usar. Propiedades:
  • value: el valor que se enviará si se elige esa opción
  • selected: opción preseleccionada por defecto
  • disable: muestra la opción pero no se puede seleccionar

<textarea>

Permite al usuario escribir texto en varias líneas, como para mensajes largos, descripciones, etc. Propiedades:
  • rows: número de filas visibles
  • cols: número de columnas visibles
  • disable
  • name
  • placeholder
  • required
